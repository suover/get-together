# 👥 GetTogether

**다양한 활동에서 함께할 사람을 찾고 소통할 수 있는 웹 커뮤니티 플랫폼**

사용자가 함께 활동할 사람을 찾거나 소통할 수 있는 서비스입니다.

## 📌 **프로젝트 소개**

### 🌟 **프로젝트 주제**

### GetTogether

함께할 사람을 찾고, 소통할 수 있는 웹 커뮤니티 플랫폼

### 💡 **개발 동기**

사람들이 공부, 취미, 운동 등 다양한 활동에서 함께할 사람을 쉽게 찾을 수 있도록 지원하는 플랫폼을 만들고자 기획했습니다.

### ⚙️ **핵심 기능**

1. **회원 시스템**: 사용자는 회원가입 후 로그인하여 게시글 작성, 댓글 작성, 좋아요 등의 서비스를 이용할 수 있습니다.
2. **게시글 작성**: 사용자는 활동에 대한 게시글을 작성하여, 함께할 사람을 찾고 자신의 활동을 공유할 수 있습니다.
3. **댓글 작성**: 게시글에 댓글을 달아 다른 사용자와 의견을 나누고 소통할 수 있습니다.
4. **좋아요**: 게시글에 좋아요를 눌러, 사용자들이 관심 있는 주제에 대해 피드백을 주고받을 수 있습니다.
5. **검색**: 사용자는 원하는 검색어를 입력하여 게시글을 쉽게 찾을 수 있습니다.

### 🔧 **개발 환경**

- Kotiln
- Spring Boot
- JPA
- Thymeleaf
- Gradle
- MySQL
- IntelliJ

### 조회 REST API 및 테스트 코드
이번 미션에서는 게시글 조회 REST API를 개발하고, 이를 검증하기 위한 테스트 코드를 작성했습니다.  
API는 전체 게시글 조회와 특정 게시글 조회 두 가지 기능을 제공합니다.

### API 엔드포인트 설명

### 1. 전체 게시글 조회
- **경로:** `GET /api/posts`
- **설명:** 전체 게시글을 조회하여 반환합니다.
- **응답 예시**
  ```json
    {
      "postId": 1,
      "title": "제목 1",
      "content": "내용 1",
      "userId": 1,
      "categoryId": 1,
      "viewCount": 100,
      "createdAt": "2024-10-10T10:00:00"
    }
  ```

### 2. 특정 게시글 조회
- **경로:** `GET /api/posts/{postId}`
- **설명:** 특정 `postId`에 해당하는 게시글 정보를 반환합니다.
- **응답 예시**
  ```json
    {
      "postId": 3,
      "title": "제목 3",
      "content": "내용 3",
      "userId": 1,
      "categoryId": 1,
      "viewCount": 100,
      "createdAt": "2024-10-10T10:00:00"
    }
  ```

## **[미션5] 삽입, 수정, 삭제 REST API 만들기**

### 삽입, 수정, 삭제 REST API 및 테스트 코드
이번 미션에서는 게시글 삽입, 수정, 삭제 REST API를 개발하고, 이를 검증하기 위한 테스트 코드를 작성했습니다.  
API는 게시글 작성(삽입), 수정, 삭제 세 가지 기능을 제공합니다.

### API 엔드포인트 설명

### 1. 게시글 삽입 (POST)
- **경로**: `POST /api/posts`
- **설명**: 새로운 게시글을 작성합니다.
- **응답 예시**
  ```json
    {
      "postId": 1,
      "title": "새 게시글",
      "content": "게시글 내용",
      "userId": 1,
      "categoryId": 1,
      "viewCount": 0,
      "createdAt": "2024-10-21T10:00:00"
    }
  ```

### 2. 게시글 수정 (PUT)
- **경로**: `PUT /api/posts/{postId}`
- **설명**: 특정 게시글을 수정합니다.
- **응답 예시**
  ```json
    {
      "postId": 1,
      "title": "수정된 제목",
      "content": "수정된 내용",
      "userId": 1,
      "categoryId": 1,
      "viewCount": 100,
      "createdAt": "2024-10-10T10:00:00"
    }
  ```

### 3. 게시글 삭제 (DELETE)
- **경로**: `DELETE /api/posts/{postId}`
- **설명**: 특정 게시글을 삭제합니다.
- **응답**: 상태코드 `204 No Content`

---

### 테스트 항목

### 1. 게시글 삽입 테스트
- **성공 케이스**: 유효한 데이터로 `POST /api/posts` 요청을 보내 새로운 게시글을 성공적으로 생성하는지 확인합니다.
   - **검증**: 응답 상태코드 `201 Created`, 생성된 게시글 정보 반환.

- **필수 값 누락 케이스**: 필수 값이 누락된 경우 `POST /api/posts` 요청이 유효성 검사를 통과하지 못하고 `400 Bad Request`를 반환하는지 확인합니다.
   - **검증**: 응답 상태코드 `400 Bad Request`, 유효성 검사 실패 메시지 반환.

- **잘못된 카테고리 ID 케이스**: 존재하지 않는 카테고리 ID로 게시글을 작성하려는 경우 `POST /api/posts` 요청이 `404 Not Found`를 반환하는지 확인합니다.
   - **검증**: 응답 상태코드 `404 Not Found`, 에러 메시지 반환.

### 2. 게시글 수정 테스트
- **성공 케이스**: 유효한 데이터로 `PUT /api/posts/{postId}` 요청을 보내 게시글을 성공적으로 수정하는지 확인합니다.
   - **검증**: 응답 상태코드 `200 OK`, 수정된 게시글 정보 반환.

- **존재하지 않는 ID 케이스**: 존재하지 않는 게시글 ID로 `PUT /api/posts/{postId}` 요청을 보냈을 때 `404 Not Found`가 반환되는지 확인합니다.
   - **검증**: 응답 상태코드 `404 Not Found`, 에러 메시지 반환.

- **필수 값 누락 케이스**: 필수 값이 누락된 데이터로 `PUT /api/posts/{postId}` 요청이 유효성 검사를 통과하지 못하고 `400 Bad Request`를 반환하는지 확인합니다.
   - **검증**: 응답 상태코드 `400 Bad Request`, 유효성 검사 실패 메시지 반환.

### 3. 게시글 삭제 테스트
- **성공 케이스**: 유효한 게시글 ID로 `DELETE /api/posts/{postId}` 요청을 보냈을 때, 게시글이 성공적으로 삭제되고 `204 No Content`가 반환되는지 확인합니다.
   - **검증**: 응답 상태코드 `204 No Content`, 삭제 성공 메시지 없음.

- **존재하지 않는 ID 케이스**: 존재하지 않는 게시글 ID로 `DELETE /api/posts/{postId}` 요청을 보냈을 때 `404 Not Found`가 반환되는지 확인합니다.
   - **검증**: 응답 상태코드 `404 Not Found`, 에러 메시지 반환.

- **이미 삭제된 게시글 케이스**: 이미 삭제된 게시글 ID로 다시 `DELETE /api/posts/{postId}` 요청을 보냈을 때 `404 Not Found`가 반환되는지 확인합니다.
   - **검증**: 응답 상태코드 `404 Not Found`, 에러 메시지 반환.
